## 2026-07-05 - 소스 인코딩과 저위험 오류 정리

### 작업

- CP949 C++ 소스 파일을 UTF-8 BOM으로 변환하고 한글 주석을 복구했다.
- 총알 이동의 중복 호출을 제거했다.
- 만료 총알 제거를 현재 iterator 기준 `erase`로 수정했다.
- 총알 렌더링 반복문의 signed/unsigned 비교를 정리했다.
- X 파일 리소스 배열 해제를 `delete[]`로 수정하고 텍스처 null 검사를 추가했다.
- 미사용 `CalculateDivisionPoints()`의 선형 보간식을 수정했다.

### 검증

- `Debug|x86` 빌드에 성공했다.
- `CalculateDivisionPoints()`는 선언과 정의 외 호출처가 없어 현재 게임 동작에는 영향이 없다.

### 남은 위험

- ANSI DirectX 및 Win32 API를 통한 한글 출력은 실행 환경 확인이 필요했다.
- 플레이어 충돌 처리의 맵 경계 인덱스는 별도 점검이 필요하다.

## 2026-07-19 - 빌드 및 실행 절차 보강

### 확인한 설정

- 솔루션의 `Debug|x86`은 프로젝트의 `Debug|Win32`로 매핑된다.
- 플랫폼 툴셋은 `v143`, Windows SDK 대상은 `10.0`이다.
- `Debug|Win32`는 DirectX SDK June 2010의 기본 `Include`와 `Lib\x86` 경로를 참조한다.
- 실행 파일은 `Debug/DirectX9_Maze.exe`에 생성된다.
- 텍스처와 X 파일 모델이 상대 경로로 로드되므로 저장소 루트가 실행 작업 디렉터리여야 한다.

### 문서 변경

- README에 필수 개발 환경과 DirectX SDK 경로를 기록했다.
- Visual Studio와 PowerShell에서의 빌드 및 실행 순서를 추가했다.
- 현재 검증된 구성을 `Debug|x86`으로 한정했다.
- 설정이 완성되지 않은 `x64`와 `Release` 구성을 현재 검증 범위에서 제외했다.

### 검증

- README에 기록한 MSBuild 명령으로 `Debug|x86` 빌드를 다시 수행했다.
- Visual Studio 2022 MSBuild 17.14.51에서 빌드에 성공했다.
- 빌드 결과는 경고 0개, 오류 0개이며 `Debug/DirectX9_Maze.exe` 생성을 확인했다.
- Roadmap의 `P0-04`를 완료로 변경했다.
