## 2026-07-19 - 빌드 및 실행 절차 보강

### 확인한 설정

- 솔루션의 `Debug|x86`은 프로젝트의 `Debug|Win32`로 매핑된다.
- 플랫폼 툴셋은 `v143`, Windows SDK 대상은 `10.0`이다.
- `Debug|Win32`는 DirectX SDK June 2010의 기본 `Include`와 `Lib\x86` 경로를 참조한다.
- 실행 파일은 `Debug/DirectX9_Maze.exe`에 생성된다.
- 텍스처와 X 파일 모델이 상대 경로로 로드되므로 저장소 루트가 실행 작업 디렉터리여야 한다.

### 문서 변경

- README에 필수 개발 환경과 DirectX SDK 경로를 기록했다.
- Visual Studio와 PowerShell에서의 빌드 및 실행 순서를 추가했다.
- 현재 검증된 구성을 `Debug|x86`으로 한정했다.
- 설정이 완성되지 않은 `x64`와 `Release` 구성을 현재 검증 범위에서 제외했다.

### 검증

- README에 기록한 MSBuild 명령으로 `Debug|x86` 빌드를 다시 수행했다.
- Visual Studio 2022 MSBuild 17.14.51에서 빌드에 성공했다.

## 2026-07-19 - 플레이어 충돌 인덱스 경계 분석

### 원인

- `CPlayer::Move()`는 좌우 이동 시 현재 행의 위·현재·아래 세 셀을 `nCoZ - 1 + i`로 조회한다.
- 첫 위치 `(55, 5, -65)`는 마지막 행인 `nCoZ == 13`에 해당하므로 좌우 이동 검사에서 행 14를 조회할 수 있다.
- 앞뒤 이동 시에는 현재 열의 왼쪽·현재·오른쪽 세 셀을 `nCoX - 1 + i`로 조회해, 첫 번째 열에서 열 -1을 조회할 수 있다.
- 오른쪽 경계의 열 12는 배열에 포함된 문자열 종료 문자이지만 플레이 가능한 맵 열은 아니므로 충돌 셀로 조회해서는 안 된다.

### 영향 위치

- `CPlayer.cpp:128`, `CPlayer.cpp:160`: 좌우 이동 시 행 범위 검사 없이 맵 조회
- `CPlayer.cpp:193`, `CPlayer.cpp:224`: 앞뒤 이동 시 열 범위 검사 없이 맵 조회
- 각 분기의 추가 벽 확인도 공통된 안전한 조회 방식을 사용해야 한다.

### 수정 방향

- 행과 열이 각각 `0 <= row < NUM_OF_ROW`, `0 <= column < NUM_OF_COLUMN`인지 확인하는 작은 공통 함수를 `CPlayer.cpp` 구현 영역에 추가한다.
- 모든 벽 조회를 범위 검사와 `'*'` 비교를 함께 수행하는 함수로 통일한다.
- 범위 밖 이웃 셀은 조회를 생략하고, 기존 외곽 좌표 보정 로직은 유지해 현재 게임 동작 변화 범위를 줄인다.
- 이동 및 충돌 구조 전체를 다시 작성하는 작업은 `P3-07`에서 별도로 다룬다.

### 검증 계획

- 정적 검색으로 `CPlayer::Move()`의 직접 `map[row][column]` 접근이 남지 않았는지 확인한다.
- `Debug|x86` 빌드에서 경고 0개, 오류 0개를 확인한다.
- 시작 위치에서 좌우·앞뒤 이동을 확인한다.
- 맵의 위·아래·왼쪽·오른쪽 가장자리와 모서리에서 이동해 맵 밖으로 나가지 않는지 확인한다.
- 자유시점 진입과 종료 후 저장된 플레이어 위치가 정상 복원되는지 확인한다.

### 상태

- `CPlayer.cpp`의 익명 namespace에 `bool IsWall()`을 추가했다.
- 직접 맵을 조회하던 6곳을 `IsWall()` 호출로 교체했다.
- 플레이 가능한 행과 열 범위를 벗어난 요청은 맵 배열을 읽지 않고 `false`를 반환한다.

### 검증 결과

- 정적 검색에서 `CPlayer::Move()`의 직접 `map[row][column]` 접근이 남지 않은 것을 확인했다.
- `Debug|x86` 빌드에 성공했다.
- 빌드 결과는 경고 0개, 오류 0개다.
- 실제 맵 네 가장자리와 모서리의 이동 감각은 다음 실행 시 수동으로 추가 확인한다.
- Roadmap의 `P1-05`를 완료로 변경했다.
