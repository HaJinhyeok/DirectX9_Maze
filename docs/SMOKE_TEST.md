# Smoke Test

## 목적

코드 정리 전후에 핵심 게임 흐름이 유지되는지 같은 순서로 빠르게 확인한다. 자동화되지 않은 항목은 수동으로 수행하고 결과와 근거를 실행 기록에 남긴다.

## 기준 환경

- 구성: `Debug|x86`
- 실행 파일: `Debug/DirectX9_Maze.exe`
- 작업 디렉터리: 저장소 루트
- 필수 환경: Visual Studio 2022, Windows 10 SDK, DirectX SDK June 2010

## 결과 상태

| 상태 | 의미 |
| --- | --- |
| 통과 | 기대 결과를 모두 만족함 |
| 실패 | 기대 결과와 다른 동작 또는 충돌이 발생함 |
| 보류 | 환경이나 선행 조건 때문에 판단할 수 없음 |
| 미실행 | 아직 수행하지 않음 |

## 실행 전 준비

1. 작업 중인 변경사항과 테스트 대상 커밋을 기록한다.
2. `Debug|x86`으로 전체 솔루션을 빌드한다.
3. 저장소 루트에서 `Debug/DirectX9_Maze.exe`를 실행한다.
4. 테스트 중 실패하면 즉시 테스트 ID, 재현 순서, 실제 결과와 스크린샷을 기록한다.

## 테스트 항목

| ID | 영역 | 수행 방법 | 기대 결과 |
| --- | --- | --- | --- |
| SMK-01 | 빌드 | `Debug|x86` 전체 빌드 | 경고 0개, 오류 0개로 실행 파일이 생성됨 |
| SMK-02 | 초기 화면 | 게임 실행 후 첫 화면 확인 | 미로, 벽·바닥 텍스처, 호랑이, 한글 조작 안내와 FPS가 표시됨 |
| SMK-03 | 기본 이동 | `W/A/S/D`와 방향키를 각각 입력 | 앞·뒤·좌·우 이동이 입력 방향에 맞게 동작함 |
| SMK-04 | 충돌과 경계 | 시작 위치와 맵의 네 가장자리·모서리에서 벽 방향으로 이동 | 벽을 통과하거나 맵 밖으로 나가지 않고 충돌이나 비정상 종료가 없음 |
| SMK-05 | 마우스 카메라 | 플레이 중 마우스를 상·하·좌·우로 이동 | 1인칭 시점이 회전하고 커서가 화면 중앙으로 복귀함 |
| SMK-06 | 키보드 회전 | `Q`, `E`를 각각 누름 | 플레이어 시점이 좌우로 회전함 |
| SMK-07 | 총알 | 좌클릭 후 발사체를 관찰하고 여러 번 연속 클릭 | 총알이 시선 방향으로 이동하며 여러 발이 서로 잘못 제거되지 않음 |
| SMK-08 | 낮·밤 | `1`을 두 번 누름 | 전체 조명 상태가 전환되고 두 번째 입력에서 원래 상태로 복구됨 |
| SMK-09 | 탑뷰 | `2`를 두 번 누름 | 탑뷰와 1인칭 시점이 전환되고 원래 시점으로 복구됨 |
| SMK-10 | 손전등 | 어두운 상태에서 `3`을 두 번 누름 | 플레이어 스팟 라이트가 꺼졌다 켜지며 시점 방향을 따라감 |
| SMK-11 | 자유시점 | `4`로 진입해 이동·회전한 뒤 다시 `4`를 누름 | `자유시점 ON`이 표시되고 종료 시 진입 전 위치와 시점으로 복구됨 |
| SMK-12 | 일시정지 | `Esc`를 누르고 이동을 시도한 뒤 다시 `Esc` 입력 | `PAUSE` UI가 표시되고 게임 갱신이 멈추며 해제 후 다시 동작함 |
| SMK-13 | 안내판 | `@` 안내판에 접근 | 안내판이 플레이어를 향하고 상호작용 범위 진입 시 연결된 시점 전환이 동작함 |
| SMK-14 | 클리어 | 미로의 `X` 출구에 접근 | `CLEAR`와 게임 내부 `exit` 버튼이 표시됨 |
| SMK-15 | 게임 내부 종료 | 일시정지 또는 클리어 화면에서 `exit` 버튼 클릭 | 정상 정리 경로를 거쳐 프로세스가 종료 코드 0으로 종료됨 |
| SMK-16 | 창 종료 | 타이틀 바의 닫기 버튼 클릭 | `WM_DESTROY`와 `CleanUp()`을 거쳐 프로세스가 종료 코드 0으로 종료됨 |

## 완료 판정

- `SMK-01`부터 `SMK-16`까지 결과가 기록되어야 한다.
- 실패 항목은 관련 Roadmap ID 또는 수정 커밋을 메모에 연결한다.
- 빌드, 초기 화면, 이동, 충돌, 클리어와 두 종료 경로는 릴리스 전 필수 통과 항목이다.
