# D3D_MyFPS 프로젝트 가이드

## 목적

이 프로젝트는 DirectX 9 기반 1인칭 미로 게임을 현재 관점에서 복구하고 정리해, 그래픽스와 게임 클라이언트 기본기를 설명할 수 있는 보조 포트폴리오로 만드는 것을 목표로 한다.

새로운 콘텐츠를 빠르게 늘리는 것보다 기존 구현의 안정성, 책임 분리, 리소스 소유권과 설명 가능성을 먼저 개선한다.

## 보여줄 핵심 역량

- Direct3D 9 초기화와 렌더링 루프 구성
- 1인칭 카메라와 입력 처리
- 타일 맵 기반 충돌 처리
- 프러스텀 컬링과 조명
- X 파일 모델과 텍스처 로딩
- 시간 기반 이동과 간단한 게임 오브젝트 업데이트
- 오래된 C++ 프로젝트를 분석하고 단계적으로 안정화한 과정

## 개선 원칙

1. 정상 실행되는 현재 동작을 기준선으로 보존한다.
2. 문서화와 명백한 오류 수정부터 진행한다.
3. 경계 검사와 리소스 소유권을 우선 정리한다.
4. 코딩 컨벤션은 한 번에 전체를 바꾸지 않고 작업 단위별로 통일한다.
5. 입력, 업데이트, 렌더링의 경계를 분명하게 만든 뒤 클래스를 분리한다.
6. 구조 정리가 끝나기 전에는 큰 기능을 추가하지 않는다.
7. 개선 전후의 이유와 검증 결과를 Worklog에 남긴다.

## 범위

### 우선 포함

- 빌드 및 실행 재현성
- 충돌 처리 경계 안정성
- COM 및 동적 메모리 관리
- 예약 식별자와 명명 규칙 정리
- 전역 상태 축소
- 게임 업데이트와 렌더링 분리
- 맵 데이터, 모델 로딩과 게임 오브젝트 책임 분리
- README, 실행 화면과 기술 설명 보강

### 후순위

- 미로 자동 생성
- 총알과 벽 또는 호랑이의 충돌
- 난이도와 리스폰 시스템
- 클리어 타임 기록
- 미니맵과 힌트 시스템

### 현재 범위 밖

- DirectX 9 렌더러를 최신 그래픽스 API로 전면 교체
- 프로젝트 전체를 새 엔진으로 이식
- 기존 결과물의 목적과 관계없는 대규모 콘텐츠 제작
